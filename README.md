# ⚽ WC-2026 Predictor

Telegram-агент, который присылает прогнозы на матчи ЧМ-2026, **оптимизируя ожидаемые очки (EV)**
под конкретную систему пула (5/3/2/1), а не «самый вероятный счёт». Работает на бесплатном
GitHub Actions, считает математику детерминированно на Python и постоянно проверяет сам себя
против тупых бейзлайнов.

## Как это работает

```
GitHub Actions cron (каждые 30 мин)
        │
        ▼
  run.py ── для каждого матча ЧМ:
        │
        ├─ scheduler.py   когда слать? (Киев: дневной матч → за 2ч; ночной → накануне 21:00)
        ├─ odds.py        OddsPapi → de-vig по острым книгам → рыночные λ (ожидаемые голы)
        ├─ predict.py     Opus (web_search): МАЛАЯ поправка λ (±15%) + факторы + уверенность
        ├─ model.py       Dixon–Coles → матрица счетов → EV по таблице пула → выбор счёта
        ├─ telegram.py    отправка тебе в личку
        └─ calibration.py лог прогноза + 3 бейзлайна (для самоаудита)

settle.py (раз в день) — дописывает фактические результаты и очки;
по понедельникам шлёт самоаудит: обыгрывает ли EV-агент бейзлайны.
```

## Главный принцип: не быть самоуверенным

Зашит механизмами, а не лозунгом:

1. **LLM ограничен.** Opus не выдаёт вероятности и не выбирает счёт. Он может только дать
   множитель к λ в коридоре **±15%** (`MAX_LAMBDA_ADJ`), и только при наличии источника с
   датой — иначе множитель 1.0. Счёт, EV и альтернативы считает Python.
2. **Уверенность механическая.** low/medium/high выводится из: расхождения букмекеров между
   собой, расхождения рынка с Elo (сравнение «яблок с яблоками»: Elo We vs P(хозяева)+½·P(ничья)),
   подтверждены ли составы, и зазора EV между топ-кандидатами. LLM может только **понизить**
   уверенность, не повысить.
3. **Живой out-of-sample контроль.** Каждый прогноз логируется вместе с тремя бейзлайнами на тех
   же данных: модальный счёт (тест гипотезы «EV ≠ модальный»), наивный «1:0 фавориту», счёт по
   Elo. После матчей Python (не LLM) считает очки и шлёт честный аудит. Если EV-агент не
   обыгрывает тупой бейзлайн — ты видишь это первым.

> На ~104 матчах эдж в призовом пуле мал, а дисперсия велика. Смысл обвязки — показать это в
> цифрах, а не верить агенту на слово.

## Настройка (5 минут)

1. Создай репозиторий на GitHub и залей туда содержимое этой папки.
2. **Settings → Secrets and variables → Actions → New repository secret** — добавь 4 секрета:
   | Secret | Где взять |
   |---|---|
   | `TELEGRAM_BOT_TOKEN` | бот у [@BotFather](https://t.me/BotFather) |
   | `TELEGRAM_CHAT_ID` | твой id у [@userinfobot](https://t.me/userinfobot) |
   | `ANTHROPIC_API_KEY` | console.anthropic.com (закинь немного кредитов) |
   | `ODDSPAPI_KEY` | [oddspapi.io](https://oddspapi.io/) (free-тариф) |
3. Напиши боту `/start` (иначе Telegram не даст ему писать тебе первым).
4. **Actions → predict → Run workflow** — ручной прогон для проверки. Дальше крон работает сам.

### (Опционально) Elo для кросс-проверки
Положи `state/elo.json` вида `{"Brazil": 2030, "Morocco": 1820, ...}` (рейтинги World Football
Elo). Имена команд должны совпадать с теми, что отдаёт OddsPapi (`participant1Name`). Без файла
кросс-проверка по Elo и Elo-бейзлайн просто отключаются.

## Локальный прогон / отладка

```bash
pip install -r requirements.txt
python tests/test_model.py        # математика (scoring, Dixon–Coles, EV, инверсия рынка)
python tests/test_scheduler.py    # логика времени по Киеву

# Сухой прогон без ключей и без отправки (печатает в консоль):
DRY_RUN=1 python run.py

# Посмотреть сырой ответ OddsPapi по матчу (нужен ключ):
python -m agent.odds <fixtureId>
```

## Стоимость

- OddsPapi, OpenWeather (через web_search), GitHub Actions, Telegram — **бесплатно**.
  OddsPapi free = 250 запросов/мес; агент берёт ~1 запрос на матч → весь турнир влезает.
- Anthropic API (Opus `claude-opus-4-8`, $5/$25 за 1M токенов) — **порядка $0.15–0.35 за матч,
  ~$20–35 за весь ЧМ**, плюс копейки за web-поиск. Кэширование системного промпта снижает это.

## Что проверено, а что требует твоих ключей

- ✅ **Протестировано офлайн:** scoring-правило (на примере из брифа), Dixon–Coles (сумма=1),
  EV-движок, инверсия рынка → λ (round-trip), вся логика времени, сквозной dry-run конвейера.
- ⚠️ **Проверь на первом реальном ответе:** маппинг полей OddsPapi (`odds.py`, функции
  `extract_*`) сделан строго по их докам, но это внешняя схема — `python -m agent.odds <id>`
  покажет реальный JSON, и если что-то отличается, правится только ~10 строк в `odds.py`.
- ⚠️ **Требуют живых ключей:** вызовы Anthropic и Telegram (тут сухой прогон их не трогает).

## Подстройка

- `MAX_LAMBDA_ADJ` — коридор поправки LLM (по умолчанию 0.15).
- `SHARP_BOOKS` — на каких книгах якорим fair-линию (по умолчанию pinnacle,polymarket,kalshi,bet365,draftkings).
- Окно сна/бодрствования и время отправки — в `agent/scheduler.py`.
- Частота крона — в `.github/workflows/cron.yml`.
